# モバイル完結開発ガイド 📱

このプロジェクトは、スマートフォンやタブレットだけで開発できます！
外出先でも気軽にコードを書いて、実機でプレビューを確認できます。

## 🏆 推奨方法：GitHub Web UI + Vercel

**最もシンプルで実用的な方法です！**

### 特徴
- ✅ PCは不要、スマホ・タブレットだけでOK
- ✅ GitHub Web UIで直接コード編集
- ✅ **完全無料・無制限**（時間制限なし）
- ✅ **実機プレビュー**（Vercel自動デプロイ）
- ✅ 本番環境と同じ環境でテスト
- ✅ モバイルUIの確認に最適

### 始め方（3分）

#### 1. ファイルを編集

**スマホ・タブレットのブラウザで：**

1. **GitHubリポジトリにアクセス**
   ```
   https://github.com/Martensiter/keiba-calculator
   ```

2. **編集したいファイルを開く**
   
   例：`src/app/page.tsx`
   - フォルダをタップして移動
   - ファイルをタップして開く

3. **鉛筆アイコン（✏️）をタップ**
   
   画面右上の「Edit this file」ボタン

4. **コードを編集**
   
   - ソフトキーボードで入力
   - シンタックスハイライト付き
   - 行番号も表示

5. **コミット**
   
   - 下にスクロール
   - コミットメッセージを入力
   - 「Commit changes」をタップ

#### 2. プレビュー環境で確認

**Vercelが自動的にプレビューデプロイ！**

1. **PRを確認**
   - リポジトリページに戻る
   - 「Pull requests」タブ
   - 自動生成されたPRを開く

2. **プレビューURLにアクセス**
   - Vercel botのコメントを探す
   - 「Visit Preview 👀」リンクをタップ
   - **一時的なプレビューURL**で実機確認

**この時点ではまだ本番環境には反映されていません**

#### 3. 本番環境にデプロイ（任意）

**変更を本番に反映したい場合：**

1. **PRページで「Merge pull request」をタップ**
2. **「Confirm merge」をタップ**
3. **Vercelが自動的に本番デプロイ**（30秒〜1分）
4. **本番URLで確認**
   ```
   https://keiba-calculator.vercel.app/
   ```

**実機でアプリが表示されます！** 🎉

### モバイルでの操作方法

#### GitHub Web UIでの編集

**単一ファイルの編集:**
1. ファイルを開く
2. ✏️ アイコンをタップ
3. 編集
4. コミット

**複数ファイルの編集:**
1. ファイルを開く
2. `.` キー（ピリオド）を押す
3. **github.dev** が起動（軽量VS Code）
4. 複数ファイルを編集可能
5. 左の「Source Control」からコミット

**ブランチの作成:**
1. ブランチドロップダウンをタップ
2. 「Find or create a branch」に名前を入力
3. 「Create branch」

#### デプロイの確認

**ステータス確認:**
- リポジトリページの「Actions」タブ
- ✅ 緑チェック = デプロイ成功
- 🟡 黄色点滅 = デプロイ中
- ❌ 赤バツ = エラー

**プレビューURL:**
- PRページのVercel botコメント
- または Vercel ダッシュボード

#### 便利なTips

- **github.dev起動**: URLの `github.com` を `github.dev` に変更
- **ファイル検索**: `t` キー
- **コマンドパレット**: `Ctrl+K` または画面上部の検索
- **変更差分**: PRページで確認

## トラブルシューティング

### Vercelデプロイが失敗する

**確認事項:**
1. GitHubのActionsタブでエラー確認
2. ビルドログを確認（Vercelダッシュボード）
3. `package.json`の構文エラーをチェック

**よくあるエラー:**
- TypeScriptの型エラー → ローカルで`npm run build`確認
- 依存関係の問題 → `package.json`を確認

### プレビューURLが見つからない

**方法1: GitHubから**
- PRページのVercel botコメントを確認

**方法2: Vercelから**
- https://vercel.com → プロジェクト → Deployments

**方法3: 通知を有効化**
- Vercelアプリの通知設定をON
- デプロイ完了時に通知が来る

### GitHub Web UIで編集できない

- ブランチの権限を確認
- リポジトリのオーナーか確認
- ファイルサイズが大きすぎないか（1MB以下推奨）

**代替案:**
- github.dev を使う（`.` キーまたはURLを変更）
- Codespacesを使う

### 変更が反映されない

**確認:**
1. コミットが正常に完了したか
2. Vercelがデプロイを開始したか
3. 正しいプレビューURLを開いているか

**対処:**
- ブラウザのキャッシュをクリア
- シークレットモードで開く
- 強制リロード（Shift + 更新）
