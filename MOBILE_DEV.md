# モバイル完結開発ガイド 📱

このプロジェクトは、スマートフォンやタブレットだけで開発できます！
外出先でも気軽にコードを書いて、プレビューを確認できます。

## 推奨方法：GitHub Codespaces

GitHub Codespacesを使えば、モバイルブラウザから本格的な開発環境にアクセスできます。

### 特徴
- ✅ PCは不要、スマホ・タブレットだけでOK
- ✅ フル機能のVS Codeがブラウザで動作
- ✅ リアルタイムプレビュー
- ✅ 自動保存・Git操作も可能
- ✅ 無料枠あり（月60時間まで）

### 始め方

#### 1. GitHub Codespacesを起動

**スマホ・タブレットのブラウザで：**

1. GitHubリポジトリにアクセス
   ```
   https://github.com/Martensiter/keiba-calculator
   ```

2. 緑色の「Code」ボタンをタップ

3. 「Codespaces」タブを選択

4. 「Create codespace on [ブランチ名]」をタップ

5. 数十秒〜数分で開発環境が起動します

#### 2. 開発サーバーを起動

Codespaceが起動したら、ターミナルで：

```bash
npm run dev
```

自動的にポート3000が転送され、ブラウザで開くか尋ねられます。

#### 3. プレビューを確認

- 「Open in Browser」をタップすると、別タブでアプリが開きます
- コードを変更すると、自動的にリロードされます

### モバイルでの操作方法

#### ファイルを開く
- 左サイドバーのファイルエクスプローラーからタップ

#### コードを編集
- タップしてカーソルを移動
- ソフトキーボードで入力
- 自動補完も機能します

#### Git操作
1. 左サイドバーの「Source Control」アイコンをタップ
2. 変更したファイルが表示されます
3. コミットメッセージを入力
4. 「✓」アイコンでコミット
5. 「...」メニューから「Push」を選択

#### ターミナル
- 下部のターミナルパネルでコマンド実行
- `Ctrl+C`は画面キーボードの特殊キーから

### 便利なショートカット（モバイル）

- **ファイル検索**: 上部の検索バーをタップ
- **コマンドパレット**: ⋮メニュー → Command Palette
- **マルチカーソル**: 長押ししてドラッグ
- **画面分割**: ファイルを長押しして「Split」

## 代替方法1：StackBlitz

ブラウザ上でNext.jsが動作する軽量な開発環境です。

### 始め方

1. StackBlitzにアクセス
   ```
   https://stackblitz.com
   ```

2. 「Import from GitHub」を選択

3. リポジトリURLを入力
   ```
   Martensiter/keiba-calculator
   ```

4. 自動的にプロジェクトがインポートされ、プレビューが表示されます

### 特徴
- ✅ 起動が非常に速い（数秒）
- ✅ GitHub連携が簡単
- ✅ 完全無料
- ⚠️ 一部機能に制限あり

## モバイル開発のコツ

### 1. タブレット推奨
スマホでも可能ですが、タブレット（iPad、Android tablet）の方が快適です。

### 2. 外付けキーボード
Bluetooth キーボードを使うと、PC並みの快適さになります。

### 3. 分割画面
- コードとプレビューを同時表示
- タブレットなら2画面で効率アップ

### 4. オフライン対応
- Codespacesは一度起動すれば短時間のオフラインOK
- コミット前にネットワーク接続を確認

### 5. バッテリー管理
- 開発作業はバッテリーを消費します
- 充電しながらの作業を推奨

## トラブルシューティング

### Codespacesが起動しない
- ブラウザのキャッシュをクリア
- 別のブラウザを試す（Chrome、Safari、Firefox）
- GitHub Statusページで障害情報を確認

### プレビューが表示されない
```bash
# ターミナルで確認
npm run dev

# ポート転送を確認
# 「PORTS」タブで3000番ポートが表示されているか
```

### 重い・遅い
- 不要なタブを閉じる
- 他のアプリを終了
- Codespacesのマシンタイプを変更（Settings）

### 保存されない
- Codespacesは自動保存されます
- 念のためcommit/pushを習慣化しましょう
