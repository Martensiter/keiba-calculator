このガイドに従えば、**スマホ・タブレットだけで**今すぐ開発を始められます！

## 📱 必要なもの

- スマートフォンまたはタブレット
- GitHubアカウント
- インターネット接続

**それだけ！PCは不要です。**

---

## ステップ1：ファイルを編集（1分）

### スマホ・タブレットで操作

1. **GitHubリポジトリにアクセス**
   
   ブラウザ（Chrome、Safari、Firefoxなど）で以下にアクセス：
   ```
   https://github.com/Martensiter/keiba-calculator
   ```

2. **編集したいファイルを開く**
   
   例として `src/app/page.tsx` を開きます：
   - 画面上の「**src**」フォルダをタップ
   - 「**app**」フォルダをタップ
   - 「**page.tsx**」ファイルをタップ

3. **編集モードに入る**
   
   画面右上の **✏️ 鉛筆アイコン**（Edit this file）をタップ

4. **コードを編集してみる**
   
   36行目あたりを探して：
   ```typescript
   const [betType, setBetType] = useState<BetType>('馬連');
   ```
   
   これを以下に変更：
   ```typescript
   const [betType, setBetType] = useState<BetType>('単勝');
   ```
   
   ソフトキーボードで入力できます！

5. **コミット（保存）する**
   
   - 画面を下にスクロール
   - 「Commit message」に「初期選択を単勝に変更」と入力
   - 緑色の「**Commit changes**」ボタンをタップ

## 🆘 困ったときは

### プレビューが見つからない

**確認:**
1. コミットは完了してる？（GitHubで確認）
2. Vercelはデプロイ中？（PRのチェックマーク確認）
3. 正しいURLを開いてる？

**対処:**
- Vercel ダッシュボードから直接アクセス
- GitHubのActionsタブでステータス確認

### デプロイが失敗した

**確認:**
- GitHubの「Actions」タブでエラーログ確認
- Vercelのビルドログを確認

**よくある原因:**
- タイプミス（構文エラー）
- 閉じ括弧忘れ

**対処:**
- 変更を元に戻す（Revert）
- エラーメッセージをよく読む

### 編集できない

**確認:**
- リポジトリの権限はある？
- ファイルが大きすぎない？（1MB以下推奨）

**代替案:**
- github.dev を使う（`.` キー）
- Codespacesを使う

### わからないことがある
- 📱 `MOBILE_DEV.md` - モバイル開発詳細ガイド
- 📖 `README.md` - プロジェクト概要
- 🔧 `CLAUDE.md` - 技術仕様

## 🎯 よくある質問

**Q: 料金はかかりますか？**

A: **完全無料です！**
- GitHub: 無料
- Vercel（Hobbyプラン）: 無料
- この開発方法なら追加料金ゼロ

**Q: スマホだけで本格的な開発ができますか？**

A: できます！ただし快適度は：
- 🏆 タブレット + キーボード（最高）
- ⭐ タブレット単体（快適）
- ✓ スマホ（軽微な修正向け）

**Q: オフラインでも使えますか？**

A: いいえ、オンライン環境が必要です。
- GitHub Web UIはオンライン必須
- Vercelのデプロイもオンライン必須
- カフェのWi-Fi等でOK

**Q: 変更が消えることはありますか？**

A: GitHubにコミットすれば消えません。
- コミット前に接続が切れた場合、ブラウザに一時保存される
- こまめにコミットする習慣を！

**Q: 複雑な編集もできますか？**

A: 簡単な編集はGitHub Web UIで、複雑な作業は：
- github.dev（`.` キーで起動）
- GitHub Codespaces（本格開発）
を使い分けましょう。

**Q: PCでも開発できますか？**

A: もちろん！`README.md`の通常セットアップを参照してください。
